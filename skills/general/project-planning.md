# Project Planning

> Update this file with your team's planning methodology and tools.

## Planning Process

When starting a new feature or project:

1. **Define scope**: Write a clear one-paragraph summary of what will be built and why
2. **Break down tasks**: Create discrete, implementable units of work
3. **Estimate effort**: Use T-shirt sizes (S/M/L/XL) rather than hours
4. **Identify dependencies**: Note any blockers or prerequisites
5. **Set milestones**: Define checkpoints for progress review

## Task Structure

Each task should include:

- **Title**: Clear, actionable description (e.g., "Add user login API endpoint")
- **Context**: Why this task exists and what it enables
- **Acceptance criteria**: How to know when it's done
- **Dependencies**: What must be completed first

## Estimation Guide

| Size | Scope | Example |
|------|-------|---------|
| **S** | Single file change, < 1 hour | Fix a bug, add a config value |
| **M** | Few files, few hours | Add an API endpoint, create a component |
| **L** | Multiple files/systems, 1-2 days | New feature with frontend + backend |
| **XL** | Architecture change, 3+ days | New service, major refactor |

## AI-Assisted Planning

When using an AI assistant to plan work:

1. Start by describing the goal and constraints
2. Ask the AI to break down the work into tasks
3. Review and adjust the task list
4. Use the AI to implement tasks one by one, reviewing each before moving on
